## Directory Purposes

**src/**
- Purpose: All application source code
- Contains: React component, entry point, global styles
- Key files: `App.tsx` (main component), `main.tsx` (entry), `index.css` (styles)
- Subdirectories: None (flat structure)

**Root directory**
- Purpose: Configuration files, assets, documentation
- Contains: Build configs, HTML template, images, metadata
- Key files: `vite.config.ts`, `tsconfig.json`, `package.json`

## Key File Locations

**Entry Points:**
- `index.html`: HTML template with font loading and #root div
- `src/main.tsx`: React DOM initialization
- `src/App.tsx`: Main application component

**Configuration:**
- `vite.config.ts`: Build tool config (React plugin, Tailwind, path aliases, env vars)
- `tsconfig.json`: TypeScript compiler options (ES2022, ESNext modules, react-jsx)
- `package.json`: Dependencies, scripts (dev, build, preview, clean, lint)
- `.env.example`: Environment variable documentation (GEMINI_API_KEY, APP_URL)

**Core Logic:**
- `src/App.tsx`: Entire application UI, state management, event handlers

**Testing:**
- Not present (no test files or test configuration)

**Documentation:**
- `README.md`: Setup instructions and prerequisites
- `metadata.json`: AI Studio project metadata
- `웹사이트 기획서 양식_K-MEN.pdf`: Website planning document (Korean)

## Naming Conventions

**Files:**
- PascalCase for React components: `App.tsx`
- lowercase for entry/utility: `main.tsx`, `index.css`
- kebab-case for config: `vite.config.ts`
- lowercase for standard: `package.json`, `tsconfig.json`

**Directories:**
- lowercase: `src/`
- dot-prefix for system: `.planning/`

**Special Patterns:**
- `.env.example` for environment template
- No barrel files (index.ts) - single component app

## Where to Add New Code

**New Component:**
- Implementation: `src/components/{ComponentName}.tsx` (directory to create)
- Types: `src/types/{name}.ts` (directory to create)
- Tests: `src/components/{ComponentName}.test.tsx` (directory to create)

**New Section/Page:**
- Implementation: `src/sections/{SectionName}.tsx` (directory to create)
- Currently all sections are inline in `src/App.tsx`

**New Hook:**
- Implementation: `src/hooks/use{HookName}.ts` (directory to create)
- Currently hooks are inline in `src/App.tsx`

**New Data/Constants:**
- Implementation: `src/data/{name}.ts` (directory to create)
- Currently member data is inline in `src/App.tsx`

**Utilities:**
- Shared helpers: `src/utils/{name}.ts` (directory to create)
- No utilities exist yet
